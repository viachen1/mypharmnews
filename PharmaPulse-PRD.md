## 1. 产品概述

### 1.1 产品定位

**PharmaPulse** 是一款面向医药行业从业者、投资人及研究人员的每日新闻聚合与推送产品。通过自动抓取全球主流医药行业资讯，经过分类、过滤、AI摘要处理后，以结构化的网页形式呈现，并支持每日邮件推送。

### 1.2 核心价值

- **省时**：用户无需手动浏览多个来源，一站获取全球医药动向
- **精准**：按政策监管、临床研究、企业动态、市场动态四维分类
- **及时**：每天早8点自动更新，保持信息新鲜度
- **可读**：提供 AI 生成的中文摘要，降低英文阅读门槛

### 1.3 产品形态

- **主体**：响应式网页（Web）
- **推送**：每日邮件日报（可订阅）
- **扩展**：RSS Feed 输出

## 2. 目标用户

### 主要用户

| 用户类型 | 使用场景 | 核心诉求 |
|---------|---------|---------|
| 医药行业从业者 | 上班前快速了解行业动态 | 信息全、分类准、中文摘要 |
| 医药投资人/分析师 | 监控企业并购、药物审批动态 | 重要事件不遗漏、可归档检索 |
| 医学研究人员 | 跟踪临床试验进展、监管政策 | 学术来源可靠、附原文链接 |
| 医药企业市场部 | 竞品监控、市场情报收集 | 按公司/品类可筛选 |

### 用户画像（核心）

- **年龄：** 28–50岁
- **背景：** 医药/生物/投资/咨询行业
- **语言：** 中英文双语（优先中文摘要+英文原文）
- **使用频率：** 每日，平均阅读时间 10–20 分钟

## 3. 核心功能需求

### 3.1 功能优先级矩阵

| 优先级 | 功能模块 | 说明 |
|-------|---------|------|
| P0（MVP必须） | 新闻聚合展示 | 按日期展示当天新闻列表 |
| P0 | 分类筛选 | 政策监管 / 临床研究 / 企业动态 / 市场动态 |
| P0 | 中文摘要 | AI 生成每条新闻的中文摘要（2–4句） |
| P0 | 原文跳转 | 每条新闻附来源媒体和原文链接 |
| P1 | 邮件日报订阅 | 用户填邮箱，每天早8点收到日报 |
| P1 | 历史归档 | 按日期查看过去30天新闻 |
| P1 | 重要度标记 | 高/中/低三级，高级条目置顶高亮 |
| P2 | 关键词搜索 | 在当天或历史新闻中搜索 |
| P2 | 自定义关键词订阅 | 用户设定感兴趣词，只推送匹配内容 |
| P2 | 公司/药物标签 | 自动识别文中公司名和药物名，可点击筛选 |

### 3.2 新闻聚合详细需求

**数据处理流程：**

```
抓取原始数据
    → 去重（标题相似度 > 80% 视为重复，保留最早来源）
    → 关键词过滤（必须包含医药相关词）
    → 自动分类（规则匹配 + AI 分类）
    → AI 生成中文摘要
    → 重要度评分
    → 写入存储
    → 刷新网页
```

**分类规则（关键词触发）：**

| 分类 | 触发关键词 |
|-----|-----------|
| 政策监管 | FDA, EMA, regulatory, approval, recall, ban, NMPA, 监管, 审批 |
| 临床研究 | Phase 1/2/3, clinical trial, efficacy, safety, placebo, endpoint |
| 企业动态 | acquisition, merger, partnership, IPO, CEO, earnings, layoff |
| 市场动态 | launch, patent, generic, revenue, market share, pricing |

**重要度评分规则：**

- 🔴 **高**：FDA/EMA 正式批准或拒绝；重大并购（>$10亿）；三期临床关键数据
- 🟡 **中**：二期临床结果；企业季报；新的合作协议
- ⚪ **低**：行业评论、分析报告、一般性资讯

### 3.3 网页展示详细需求

**首页布局：**

```
顶部导航栏
├── Logo（PharmaPulse）
├── 日期选择器
└── 订阅入口

内容区
├── 今日摘要（总条数 + 各分类数量统计）
├── 分类 Tab：[全部] [政策监管] [临床研究] [企业动态] [市场动态]
└── 新闻卡片列表
    ├── 重要度标记（颜色圆点）
    ├── 标题（可点击跳转原文）
    ├── 来源媒体 + 发布时间
    ├── 中文摘要（默认折叠，点击展开）
    └── 标签（分类 + 公司名/药物名）

底部
├── 历史归档（日历或列表）

```

**新闻卡片交互：**
- 默认显示标题 + 来源 + 摘要首句
- 点击「查看摘要」展开全部中文摘要
- 点击标题直接跳转原文（新标签页）
- 标签可点击，触发同标签内容筛选

## 4. PubMed 文献检索模块

> **v1.1 新增** — 在新闻聚合基础上，补充 PubMed 学术文献检索能力，满足研究人员对学术来源的需求。该模块作为独立页面存在，不影响原有新闻功能。

### 4.1 模块定位

| 维度 | 说明 |
|------|------|
| **数据来源** | PubMed E-utilities API（NCBI 官方，免费，无需注册） |
| **用户场景** | 在看完行业新闻后，快速检索相关学术文献，获取一手研究数据 |
| **与新闻模块关系** | 互补——新闻模块聚焦行业动态，文献模块聚焦学术研究，共同构成「信息+知识」双轨 |
| **无账号设计** | 全模块无需登录，无收藏/标签，纯工具型使用体验 |

### 4.2 功能清单

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **P0** | 🔍 关键词搜索 | 输入关键词，返回 PubMed 文献列表，支持防抖（300ms） |
| **P0** | 📄 文献列表展示 | 展示标题、作者、期刊、发表日期、摘要（可折叠展开），分页浏览 |
| **P0** | 📖 文献详情 | 完整展示摘要全文、MeSH 术语、作者机构、DOI 链接、PubMed 原文链接 |
| **P1** | 🔢 高级搜索与筛选 | 时间范围（mindate/maxdate）、文章类型（综述/原著）、排序方式（相关度/最新） |
| **P1** | 📋 引用格式复制 | 文献详情页一键复制 APA / Vancouver 格式引用（纯前端生成，无需后端） |
| **P1** | 🌓 暗色模式 | 与全站统一，`localStorage` 记住偏好，护眼阅读 |
| **P2** | 📊 发文趋势图 | 搜索结果页下方展示该关键词近 10 年每年发文量折线图 |

### 4.3 PubMed API 接入方案

**调用的三个核心接口：**

| 接口 | 作用 | 输入 | 输出 |
|------|------|------|------|
| `esearch` | 关键词搜索，返回文章 ID 列表 | 查询词 + 分页参数 | PMID 列表 + 总数 |
| `esummary` | 批量获取轻量摘要信息（列表页用） | PMID 列表 | 标题/期刊/日期/作者 |
| `efetch` | 获取完整文章详情（详情页用） | 单个 PMID | 完整 XML（含摘要、MeSH、元数据） |

**调用流程：**

```
用户输入关键词
    │
    ▼
esearch(term, retmax=20, retstart=0)
    │  → 返回 {count: 总数, idlist: [PMID...]}
    ▼
esummary(id=id1,id2,...id20)
    │  → 返回列表页所需的轻量信息（快速展示）
    ▼
用户点击某篇文章
    │
    ▼
efetch(id=pmid, retmode=xml)
    │  → 返回完整文章详情（含摘要全文、MeSH 词、作者机构等）
```

**关键约束与应对：**

| 约束 | 说明 | 应对策略 |
|------|------|----------|
| 频率限制 | 无 Key: 3次/秒，有 Key: 10次/秒 | 申请免费 API Key，后端请求队列控制（≤3 并发） |
| 返回格式 | `efetch` 默认返回 XML | 后端用 Python `xml.etree.ElementTree` 解析转 JSON |
| CORS | NCBI 不保证浏览器直连 CORS | 后端代理，前端不直连 NCBI |
| 缓存 | 相同查询频繁重复 | 后端内存缓存（5 分钟 TTL），减少对 NCBI 的请求量 |

### 4.4 后端 API 接口设计

在现有 `server.py` 中新增以下路由：

| 方法 | 路径 | 功能 |
|------|------|------|
| `GET` | `/api/pubmed/search` | 搜索文献（代理 esearch + esummary），参数：`q`, `page`, `sort`, `date_from`, `date_to`, `article_type` |
| `GET` | `/api/pubmed/article/<pmid>` | 获取文献详情（代理 efetch，XML → JSON） |
| `GET` | `/api/pubmed/trends` | 获取趋势数据，参数：`q`，返回近 10 年每年发文量 |

### 4.5 前端页面设计

**新增页面：** `web/literature.html`

**页面布局：**

```
顶部导航栏（复用全站样式，加「文献检索」入口）

搜索区
├── 关键词输入框（居中，带搜索按钮）
└── 高级搜索折叠面板（时间范围 / 文章类型 / 排序方式）

├── 结果数量提示（共 X 篇，显示第 Y-Z 篇）
├── 文献卡片列表
│   ├── 标题（加粗，可点击展开详情）
│   ├── 作者 · 期刊 · 发表年份
│   ├── 摘要（默认折叠，点击展开，带渐入动画）
│   └── [查看 PubMed] [复制引用] 快捷操作
└── 分页器（上一页 / 下一页 / 跳页）

趋势图区（P2，搜索完成后自动渲染）
└── 该关键词近 10 年发文量折线图（Chart.js）
```

**文献详情展示（弹窗或侧滑面板）：**

```
标题（大字）
作者列表 · 期刊名 · 卷期 · 发表日期
━━━━━━━━━━━━━━━━━━
摘要全文（完整段落）
━━━━━━━━━━━━━━━━━━
MeSH 术语（标签形式，可点击发起新搜索）
DOI: 10.xxxx/xxx  [外链 →]
PubMed ID: XXXXXXXX  [在 PubMed 查看 →]
━━━━━━━━━━━━━━━━━━
引用格式
[APA]  Author et al. (2026). Title. Journal, Vol(Issue), Pages.  [复制]
[Vancouver]  Author. Title. Journal. 2026;Vol(Issue):Pages.  [复制]
```

### 4.6 高级搜索参数映射

| 筛选项 | 对应 PubMed 参数 | 示例 |
|--------|----------------|------|
| 时间范围 | `datetype=pdat&mindate=YYYY/MM/DD&maxdate=YYYY/MM/DD` | 近两年 |
| 文章类型 | `filter=pubt.review` / `filter=pubt.clinicaltrial` | 只看综述 |
| 语言 | `filter=lang.english` | 只看英文 |
| 排序 | `sort=relevance`（相关度） / `sort=date`（最新） | — |

### 4.7 发文趋势图实现方式

- 用户搜索关键词后，后端对每个年份（近 10 年）各发一次 `esearch`，取 `count` 值
- 返回格式：`{"2016": 1200, "2017": 1450, ..., "2025": 3200}`
- 前端用 Chart.js（CDN 引入）渲染折线图，X 轴年份，Y 轴文章数
- 可复用全站的深蓝配色

## 5. 数据来源规划

### 5.1 API 来源

| 来源 | 类型 | 免费额度 | 内容特点 |
|-----|------|---------|---------|
| NewsAPI.org | REST API | 100次/天 | 全球主流媒体，支持 `q=pharma` 过滤 |
| GNews API | REST API | 100次/天 | 支持中文报道 |
| Perigon | REST API | 500次/月 | 结构化好，事件聚合能力强 |

### 5.2 免费 RSS 来源（无限量）

| 来源 | RSS 地址 | 内容 |
|-----|---------|------|
| FDA 官方 | `https://www.fda.gov/about-fda/contact-fda/stay-informed/rss-feeds/` | 审批/召回/公告 |
| EMA 欧洲药管局 | `https://www.ema.europa.eu/en/news-events/rss-feeds` | 欧洲药物动态 |
| STAT News | `https://www.statnews.com/feed/` | 专业医药新闻 |
| BioPharma Dive | `https://www.biopharmadive.com/feeds/news/` | 生物医药聚焦 |
| FiercePharma | `https://www.fiercepharma.com/rss/xml` | 行业快讯 |
| PubMed 新发表 | `https://pubmed.ncbi.nlm.nih.gov/rss/search/` | 学术研究 |
| Reuters Health | `https://feeds.reuters.com/reuters/healthNews` | 综合健康医药 |

### 5.3 关键词过滤策略

**必须包含（OR 关系）：**

```
pharma | pharmaceutical | drug | FDA | EMA | NMPA | 
biotech | vaccine | clinical trial | drug approval | 
drug recall | pipeline | oncology | biologic
```

**排除词（降低噪音）：**

```
recreational drug | drug abuse | street drug | illegal drug
```

## 8. 推送功能

### 8.1 邮件日报

**触发时机：** 每天早8:00（UTC+8）数据更新完成后自动发送

**邮件结构：**

```
主题：💊 PharmaPulse 日报 | 2026-04-15 | 今日 38 条资讯
```

## 10. 技术栈选型

### MVP 阶段（零成本）

| 层级 | 技术 | 理由 |
|-----|------|------|
| 数据抓取 | Python 3.11 + `requests` + `feedparser` | 轻量，RSS解析成熟 |
| AI 摘要 | OpenAI GPT-4o-mini API | 成本极低（约$0.002/条） |
| 定时任务 | GitHub Actions | 免费，Cron支持好 |
| 存储 | JSON 文件 + Git | 零成本，天然版本控制 |
| 前端 | 纯 HTML + Tailwind CSS + Vanilla JS | 无需构建工具，GitHub Pages可直接托管 |
| 托管 | GitHub Pages | 完全免费 |
| 推送 | Resend + GitHub Actions | 3000封/月免费 |
| PubMed 代理 | Python `xml.etree.ElementTree` + 内存缓存 | 零依赖，XML 解析、频率控制、5分钟 TTL 缓存 |
| 趋势图 | Chart.js（CDN 引入） | 一个 `<script>` 标签，轻量，无构建需求 |

### 生产阶段（~$5/月）

| 层级 | 技术 | 理由 |
|-----|------|------|

| 前端 | Next.js 14 (App Router) | SSG + ISR，SEO友好 |
| 托管 | Vercel | 免费层够用，支持Cron Jobs |
| 推送 | Resend | 同上 |

## 13. 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|-----|------|------|------|
| NewsAPI 免费额度不足 | 高 | 中 | RSS 来源作为主力（无限量），API 作为补充 |
| AI 摘要质量不稳定 | 中 | 中 | 固定 Prompt 模板，每周抽查，必要时人工修正 |
| 新闻来源断更或改版 | 中 | 低 | 维护多个备用来源，监控抓取成功率 |
| 邮件送达率低 | 低 | 高 | 使用 Resend（专业邮件服务），配置 SPF/DKIM |
| 内容版权纠纷 | 低 | 高 | 只展示摘要 + 原文链接，不转载全文 |
| LLM API 成本超支 | 低 | 中 | 使用 GPT-4o-mini（极低成本），设置每日用量上限 |
| PubMed API 频率限制触发 | 低 | 中 | 申请免费 API Key，后端请求队列（≤3 并发），5分钟缓存 |
| PubMed XML 解析异常 | 低 | 低 | 容错处理，缺失字段给默认值，不影响整体展示 |
