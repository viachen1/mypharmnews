## 1. 产品概述

### 1.1 产品定位

**PharmaPulse** 是一款面向医药行业从业者、投资人及研究人员的每日新闻聚合与推送产品。通过自动抓取全球主流医药行业资讯，经过分类、过滤、AI摘要处理后，以结构化的网页形式呈现，并支持每日邮件推送。

### 1.2 核心价值

- **省时**：用户无需手动浏览多个来源，一站获取全球医药动向
- **精准**：按政策监管、临床研究、企业动态、市场动态四维分类
- **及时**：每天早8点自动更新，保持信息新鲜度
- **可读**：提供 AI 生成的中文摘要，降低英文阅读门槛

### 1.3 产品形态

- **主体**：响应式网页（Web）
- **推送**：每日邮件日报（可订阅）
- **扩展**：RSS Feed 输出

## 2. 目标用户

### 主要用户

| 用户类型 | 使用场景 | 核心诉求 |
|---------|---------|---------|
| 医药行业从业者 | 上班前快速了解行业动态 | 信息全、分类准、中文摘要 |
| 医药投资人/分析师 | 监控企业并购、药物审批动态 | 重要事件不遗漏、可归档检索 |

- **年龄：** 28–50岁
- **背景：** 医药/生物/投资/咨询行业
- **语言：** 中英文双语（优先中文摘要+英文原文）
- **使用频率：** 每日，平均阅读时间 10–20 分钟

## 3. 核心功能需求

### 3.1 功能优先级矩阵

| 优先级 | 功能模块 | 说明 |
|-------|---------|------|
| P0（MVP必须） | 新闻聚合展示 | 按日期展示当天新闻列表 |
| P0 | 分类筛选 | 政策监管 / 临床研究 / 企业动态 / 市场动态 |
| P0 | 中文摘要 | AI 生成每条新闻的中文摘要（2–4句） |
| P0 | 原文跳转 | 每条新闻附来源媒体和原文链接 |
| P1 | 邮件日报订阅 | 用户填邮箱，每天早8点收到日报 |
| P1 | 历史归档 | 按日期查看过去30天新闻 |
| P1 | 重要度标记 | 高/中/低三级，高级条目置顶高亮 |
| P2 | 关键词搜索 | 在当天或历史新闻中搜索 |
| P2 | 自定义关键词订阅 | 用户设定感兴趣词，只推送匹配内容 |
| P2 | 公司/药物标签 | 自动识别文中公司名和药物名，可点击筛选 |

### 3.2 新闻聚合详细需求

**数据处理流程：**

```
抓取原始数据
    → 去重（标题相似度 > 80% 视为重复，保留最早来源）
    → 关键词过滤（必须包含医药相关词）
    → 自动分类（规则匹配 + AI 分类）
    → AI 生成中文摘要
    → 重要度评分
    → 写入存储
    → 刷新网页
```

**分类规则（关键词触发）：**

| 分类 | 触发关键词 |
|-----|-----------|
| 政策监管 | FDA, EMA, regulatory, approval, recall, ban, NMPA, 监管, 审批 |
| 临床研究 | Phase 1/2/3, clinical trial, efficacy, safety, placebo, endpoint |
| 企业动态 | acquisition, merger, partnership, IPO, CEO, earnings, layoff |
| 市场动态 | launch, patent, generic, revenue, market share, pricing |

**重要度评分规则：**

- 🔴 **高**：FDA/EMA 正式批准或拒绝；重大并购（>$10亿）；三期临床关键数据
- 🟡 **中**：二期临床结果；企业季报；新的合作协议
- ⚪ **低**：行业评论、分析报告、一般性资讯

### 3.3 网页展示详细需求

**首页布局：**

```
顶部导航栏
├── Logo（PharmaPulse）
├── 日期选择器
└── 订阅入口

内容区
├── 今日摘要（总条数 + 各分类数量统计）
├── 分类 Tab：[全部] [政策监管] [临床研究] [企业动态] [市场动态]
└── 新闻卡片列表
    ├── 重要度标记（颜色圆点）
    ├── 标题（可点击跳转原文）
    ├── 来源媒体 + 发布时间
    ├── 中文摘要（默认折叠，点击展开）
    └── 标签（分类 + 公司名/药物名）

底部
├── 历史归档（日历或列表）
└── 页脚（关于 / 订阅 / RSS）
```

**新闻卡片交互：**
- 默认显示标题 + 来源 + 摘要首句
- 点击「查看摘要」展开全部中文摘要
- 点击标题直接跳转原文（新标签页）
- 标签可点击，触发同标签内容筛选

## 4. 数据来源规划

### 4.1 API 来源

| 来源 | 类型 | 免费额度 | 内容特点 |
|-----|------|---------|---------|
| NewsAPI.org | REST API | 100次/天 | 全球主流媒体，支持 `q=pharma` 过滤 |
| GNews API | REST API | 100次/天 | 支持中文报道 |
| Perigon | REST API | 500次/月 | 结构化好，事件聚合能力强 |

### 4.2 免费 RSS 来源（无限量）

| 来源 | RSS 地址 | 内容 |
|-----|---------|------|
| FDA 官方 | `https://www.fda.gov/about-fda/contact-fda/stay-informed/rss-feeds/` | 审批/召回/公告 |
| EMA 欧洲药管局 | `https://www.ema.europa.eu/en/news-events/rss-feeds` | 欧洲药物动态 |
| STAT News | `https://www.statnews.com/feed/` | 专业医药新闻 |
| BioPharma Dive | `https://www.biopharmadive.com/feeds/news/` | 生物医药聚焦 |
| FiercePharma | `https://www.fiercepharma.com/rss/xml` | 行业快讯 |
| PubMed 新发表 | `https://pubmed.ncbi.nlm.nih.gov/rss/search/` | 学术研究 |
| Reuters Health | `https://feeds.reuters.com/reuters/healthNews` | 综合健康医药 |

### 4.3 关键词过滤策略

**必须包含（OR 关系）：**

```
pharma | pharmaceutical | drug | FDA | EMA | NMPA | 
biotech | vaccine | clinical trial | drug approval | 
drug recall | pipeline | oncology | biologic
```

**排除词（降低噪音）：**

```
recreational drug | drug abuse | street drug | illegal drug
```

## 5. 系统架构

### 5.1 整体架构图

```
┌─────────────────────────────────────────────────────┐
│              定时调度层（每天 08:00 UTC+8）            │
│              GitHub Actions Cron Job                 │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│                   数据采集层                          │
│  NewsAPI + GNews + RSS Feeds（FDA/EMA/STAT等）       │
│  Python: requests + feedparser                       │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│                   数据处理层                          │
│  1. 去重（标题哈希 + 相似度）                          │
│  2. 关键词过滤                                        │
│  3. 规则分类 + AI 分类（Claude/GPT API）              │
│  4. AI 中文摘要生成                                   │
│  5. 重要度评分                                        │
└──────────────────────┬──────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│                   存储层                              │
│  MVP：JSON 文件（存入 GitHub 仓库）                    │
│  生产：Supabase PostgreSQL（免费 500MB）              │
└──────────────────────┬──────────────────────────────┘
                       ↓
          ┌────────────┴────────────┐
          ↓                         ↓
┌──────────────────┐     ┌──────────────────────┐
│    展示层         │     │      推送层            │
│  Next.js 网页    │     │  邮件日报（Resend）    │
│  Vercel 托管     │     │  RSS Feed 自动生成    │
└──────────────────┘     └──────────────────────┘
```

### 5.2 MVP 简化版架构（零成本）

```
GitHub Actions（定时触发）
    → Python 脚本（抓取 + 处理）
    → 生成 data/YYYY-MM-DD.json
    → Commit 到仓库
    → GitHub Pages 自动更新静态网页
```

## 6. 页面与交互设计

### 6.1 页面清单

| 页面 | 路由 | 说明 |
|-----|------|------|
| 首页（今日新闻） | `/` | 当天聚合新闻，默认展示全部分类 |

**第一屏（核心信息）：**
- 今日日期 + 新闻总数
- 分类 Tab 切换

**第二屏（正文列表）：**
- 中/低重要度新闻卡片
- 每卡片：标题 + 来源 + 时间 + 摘要首句 + 展开按钮

**侧边栏（可选，桌面端）：**
- 今日数据统计（各分类条数）
- 热门公司/药物标签云
- 订阅入口

### 6.3 新闻卡片设计规范

```
┌────────────────────────────────────────────┐
│ 🔴  [政策监管]  [Pfizer]                    │  ← 重要度 + 分类标签
```

- **重要度色：** 🔴 `#E53E3E` / 🟡 `#D69E2E` / ⚪ `#CBD5E0`

## 7. 推送功能

### 7.1 邮件日报

**触发时机：** 每天早8:00（UTC+8）数据更新完成后自动发送

**邮件结构：**

```
主题：💊 PharmaPulse 日报 | 2026-04-15 | 今日 38 条资讯

[今日重要事件]（仅展示 🔴 高重要度）
1. FDA 批准 XXX 药物 → 摘要（60字） [阅读原文]
2. ...

[政策监管]（N条）
...

[临床研究]（N条）
...

[企业动态]（N条）
...

[查看完整日报] → 网页链接
[退订] → 一键退订链接
```

**发送服务：** Resend（每月 3000 封免费）

### 7.2 RSS Feed

- 地址：`/rss.xml`
- 内容：最近 30 天新闻
- 格式：标准 RSS 2.0，含中文摘要
- 更新频率：每日同步

### 7.3 订阅管理

| 功能 | 说明 |
|-----|------|
| 订阅 | 填写邮箱 → 发送确认邮件 → 点击确认激活 |
| 退订 | 每封邮件底部一键退订，无需登录 |
| 分类偏好 | 用户可选择只接收某几类的推送（P2） |

## 8. 非功能性需求

### 8.1 性能

| 指标 | 目标值 |
|-----|-------|
| 首页加载时间 | < 2秒（静态页面） |
| 数据更新延迟 | 每天8:00前完成，用户访问时数据已就绪 |
| 可用性 | 99%（借助 Vercel/GitHub Pages 基础设施） |

### 8.2 数据质量

| 指标 | 目标值 |
|-----|-------|
| 去重率 | 重复内容 < 5% |
| 分类准确率 | > 90%（人工抽查验证） |
| 摘要质量 | AI生成，人工每周抽查10条评分 |
| 来源可信度 | 白名单媒体来源占比 > 80% |

### 8.3 安全与合规

- 只抓取公开可访问的新闻内容（不绕过付费墙）
- 每条新闻保留原文链接，符合新闻聚合类网站惯例
- 用户邮箱数据仅用于推送，不对外分享
- 隐私政策页面说明数据使用方式

## 9. 技术栈选型

### MVP 阶段（零成本）

| 层级 | 技术 | 理由 |
|-----|------|------|
| 数据抓取 | Python 3.11 + `requests` + `feedparser` | 轻量，RSS解析成熟 |
| AI 摘要 | OpenAI GPT-4o-mini API | 成本极低（约$0.002/条） |
| 定时任务 | GitHub Actions | 免费，Cron支持好 |
| 存储 | JSON 文件 + Git | 零成本，天然版本控制 |
| 前端 | 纯 HTML + Tailwind CSS + Vanilla JS | 无需构建工具，GitHub Pages可直接托管 |
| 托管 | GitHub Pages | 完全免费 |
| 推送 | Resend + GitHub Actions | 3000封/月免费 |

### 生产阶段（~$5/月）

| 层级 | 技术 | 理由 |
|-----|------|------|
| 数据抓取 | Python + Node.js | 更灵活 |
| 存储 | Supabase PostgreSQL | 免费500MB，支持全文搜索 |
| 前端 | Next.js 14 (App Router) | SSG + ISR，SEO友好 |
| 托管 | Vercel | 免费层够用，支持Cron Jobs |
| 推送 | Resend | 同上 |

## 12. 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|-----|------|------|------|
| NewsAPI 免费额度不足 | 高 | 中 | RSS 来源作为主力（无限量），API 作为补充 |
| AI 摘要质量不稳定 | 中 | 中 | 固定 Prompt 模板，每周抽查，必要时人工修正 |
| 新闻来源断更或改版 | 中 | 低 | 维护多个备用来源，监控抓取成功率 |
| 邮件送达率低 | 低 | 高 | 使用 Resend（专业邮件服务），配置 SPF/DKIM |
| 内容版权纠纷 | 低 | 高 | 只展示摘要 + 原文链接，不转载全文 |
| LLM API 成本超支 | 低 | 中 | 使用 GPT-4o-mini（极低成本），设置每日用量上限 |
